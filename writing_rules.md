# กฎเหล็กสำหรับการเขียนนิยาย (Writing Iron Rules)

เอกสารนี้รวบรวมกฎข้อบังคับและมาตรฐานในการเขียนนิยายเรื่องนี้ เพื่อให้มั่นใจว่าคุณภาพและสไตล์ของงานเขียนจะเป็นไปในทิศทางเดียวกันและได้มาตรฐานสูง

## 1. กฎทั่วไป (General Rules)
*   **ความยาวต่อตอน:** ต้องมีเนื้อหา **ไม่ต่ำกว่า 2,000 คำ** (นับเฉพาะเนื้อหา ไม่รวมชื่อตอนหรือคุยกับนักอ่าน)
*   **ภาษา:** ใช้ภาษาไทยปัจจุบัน (Modern Thai) ที่อ่านง่าย กระชับ ทันสมัย ไม่ใช้สำนวนโบราณหรือลิเกเกินความจำเป็น และต้องเป็นภาษาไทยทั้งหมดเท่านั้น
*   **Scale:** เนื้อเรื่องต้องดำเนินไปในระดับ **Global Scale** มีตัวละครหลากหลายเชื้อชาติ มีเหตุการณ์ที่ส่งผลกระทบระดับโลกหรือระหว่างประเทศ
*   **มุมมองการเล่าเรื่อง (POV):** บุรุษที่ 3 (Third Person Limited) เน้นโฟกัสที่ตัวเอกเป็นหลัก แต่สามารถสลับไปมุมมองอื่นได้เมื่อจำเป็นเพื่อขยายสเกลเรื่อง
*   **Mood & Tone:**
    *   **Dark Modern Fantasy / Cyber-Occult:** บรรยากาศเมืองใหญ่ที่ซ่อนความบิดเบี้ยว ทันสมัยแต่มีความลึกลับกดดัน
    *   **Realism & High Stakes:** มีการสูญเสียจริง ตัวละครสำคัญอาจตายได้ บรรยากาศกดดันและสมจริง
    *   **Character Progression:** พระเอกจะเริ่มจากจุดต่ำสุด (Underdog) และค่อยๆ เก่งขึ้นอย่างมีเสน่ห์ มีการโชว์ความสามารถที่เหนือชั้น (Show off) เมื่อถึงเวลา

## 2. กฎการใช้ภาษา (Language & Style)
*   **บทสนทนา (Dialogue):**
    *   ใช้ภาษาพูดที่คนยุคปัจจุบันใช้จริง มีคำสแลงหรือศัพท์เฉพาะกลุ่มได้บ้างเพื่อให้ดูสมจริง
    *   หลีกเลี่ยงการใช้คำฟุ่มเฟือยในประโยคบอกเล่า
*   **การบรรยาย (Narration):**
    *   เน้น "Show, Don't Tell" ให้เห็นภาพการกระทำและความรู้สึกผ่านสถานการณ์
    *   ฉากต่อสู้ต้องบรรยายความเร็ว น้ำหนัก และผลกระทบของการโจมตีให้ชัดเจน
*   **คำทับศัพท์:**
    *   ใช้คำทับศัพท์ภาษาอังกฤษสำหรับสกิล ไอเทม หรือเทคนิคเฉพาะทางได้ แต่ต้องสะกดให้ถูกต้องตามหลักนิยมหรือราชบัณฑิตฯ (เช่น สเตตัส, ดันเจี้ยน -> เซกเตอร์, สกิล)

## 3. ข้อห้ามสำคัญ (Critical Constraints)
*   **Relationships:** พระเอกมีความสัมพันธ์ลึกซึ้งกับตัวละครหญิงหลายคนได้ (Harem/Romance) แต่ต้องมีการพัฒนาความสัมพันธ์ที่มีมิติ มีเหตุผลรองรับ และมีความผูกพันทางใจที่ชัดเจน
*   **ห้าม Copy & Paste:** ห้ามลอกเลียนแบบฉากหรือพล็อตจาก Solo Leveling หรือนิยายดังเรื่องอื่นๆ โดยตรง ต้องมีความ "บิด" (Twist) ให้เป็นเอกลักษณ์ของเรื่องนี้ (Theme: Glitch/Error)
*   **ความสมเหตุสมผล:** แม้จะเป็นแฟนตาซี แต่ตรรกะภายในเรื่อง (Internal Logic) ต้องแข็งแรง ตัวละครต้องไม่ทำอะไรโง่ๆ เพื่อขับเคลื่อนพล็อต (Plot Armor ต้องเนียน)

## 4. รูปแบบการจัดหน้า (Formatting for Web Novels)
*   **การตัดย่อหน้า:** เพื่อให้อ่านง่ายบนแพลตฟอร์ม Dek-D หรือเว็บนิยายอื่นๆ **แต่ละย่อหน้าไม่ควรยาวเกิน 3-4 บรรทัด**
*   **การขึ้นบรรทัดใหม่:** สามารถขึ้นบรรทัดใหม่สำหรับประโยคสั้นๆ หรือบทสนทนา เพื่อสร้างจังหวะในการอ่าน
*   **การเว้นวรรค:** เว้นวรรคระหว่างประโยคให้ถูกต้อง ไม่ติดกันเป็นพืด

## 5. กฎความต่อเนื่องของ Lore (Continuity Rules)
*   ใช้คำหลักของโลกให้คงเส้นคงวา: **ดีบักเกอร์, เดดเซกเตอร์, บั๊ก/กลิซ, ซอร์สแฟรกเมนต์, เดอะ เคอร์เนล, เซิร์ฟเวอร์, การซิงค์**
*   ระดับพลังให้ยึดตามลำดับ **Level 0 Null User, Level 1 User, Level 2 Moderator, Level 3 Admin, Level 4 GM, Anomaly**
*   ชื่อยาและโรคให้คงที่: **Antiviral Serum / แอนติไวรัล เซรั่ม** และ **Data Corruption Syndrome**
*   เมื่อมีคำสั่งระบบหรือข้อความจากเดอะ เคอร์เนล ให้ใช้รูปแบบหน้าต่างข้อความสั้น คม และมีนัยผิดปกติได้ แต่ต้องไม่ฟุ่มเฟือย

## 6. กฎการแยกเนื้อหาสาธารณะกับโน้ตภายใน
*   **เนื้อหาสาธารณะสำหรับหน้าเว็บอ่าน:** ใช้เฉพาะบทนิยายในโฟลเดอร์ `chapters/`
*   **โน้ตภายใน/Story Bible:** ไฟล์ `world_bible.md` และ `writing_rules.md` ใช้เป็นบันทึกอ้างอิงการเขียน ไม่ต้องนำขึ้นหน้าอ่านหลัก
