# คัมภีร์โลกและการตั้งค่า (World Bible & Settings)

**Project Code:** Glitch Reality (ชื่อชั่วคราว)
**Theme:** Urban Fantasy / System Error / Cyber-Occult

## 1. แก่นเรื่อง (Core Concept)
โลกไม่ได้ถูกรุกรานโดยปีศาจจากต่างมิติ แต่ **"ความจริง" (Reality)** ของโลกกำลัง **"พังทลาย" (Glitch)** เหมือนโปรแกรมคอมพิวเตอร์ที่โค้ดเสียหาย สิ่งที่มนุษย์เรียกว่า "ดันเจี้ยน" คือไฟล์ขยะ (Corrupted Data) ที่แทรกซึมเข้ามาในเลเยอร์ของโลกความเป็นจริง

## 2. ศัพท์เฉพาะ (Terminology)

| คำศัพท์ทั่วไป (Common Term) | คำศัพท์ในเรื่อง (Unique Term in Novel) | ความหมาย / รายละเอียด |
| :--- | :--- | :--- |
| **Hunter (ฮันเตอร์)** | **Debugger (ดีบักเกอร์)** หรือ **Fixer (ฟิกเซอร์)** | ผู้ที่มีความสามารถในการมองเห็น "โค้ด" ของโลกและแก้ไขความผิดปกติ ไร้ซึ่งพลังเวท แต่ใช้การ "เขียนทับ" หรือ "ลบ" ข้อมูล |
| **Dungeon (ดันเจี้ยน)** | **Dead Sector (เดดเซกเตอร์)** | พื้นที่ที่ข้อมูลเสียหาย โลกภายนอกจะมองไม่เห็น คนธรรมดาที่หลงเข้าไปจะถูกลบตัวตน (De-rezzed) |
| **Monster (มอนสเตอร์)** | **Bug (บั๊ก)** หรือ **Glitch (กลิซ)** | สิ่งมีชีวิตที่เกิดจากข้อมูลขยะ มีรูปร่างบิดเบี้ยว ไม่สมประกอบ เลือดเป็นสีรุ้งแตกซ่าเหมือนจอทีวีเสีย |
| **Mana (มานา)** | **Source Fragments (ซอร์สแฟรกเมนต์)** | เศษข้อมูลที่ตกหล่นเมื่อกำจัด Bug ได้ ใช้สำหรับอัปเกรดหรือ "Patch" ร่างกาย |
| **System (ระบบ)** | **The Kernel (เดอะ เคอร์เนล)** | เสียงในหัวหรือหน้าต่างข้อความที่คอยแจ้งเตือน แต่มันมักจะทำงานผิดพลาด แจ้ง Error Code แทนที่จะบอกตรงๆ |
| **Guild (กิลด์)** | **Server (เซิร์ฟเวอร์)** | กลุ่มของ Debugger ที่รวมตัวกันคุมพื้นที่เขตข้อมูลต่างๆ |
| **Awakening (การตื่นรู้)** | **Synchronization (การซิงค์)** | กระบวนการที่มนุษย์เชื่อมต่อกับ The Kernel ได้สำเร็จ |

## 3. ระดับความสามารถ (Hierarchy Matrix)
แทนที่จะเป็น Rank S-E, เราใช้ระดับสิทธิ์การเข้าถึง (Access Level):

1.  **Guest (เกสต์):** คนธรรมดา ไม่รู้เรื่องราว
2.  **User (ยูสเซอร์):** ดีบักเกอร์ระดับล่าง ทำได้แค่ต่อสู้กายภาพ ใช้เครื่องมือพื้นฐาน
3.  **Moderator (โมเดอเรเตอร์):** ดีบักเกอร์ระดับกลาง เริ่มใช้คำสั่ง (Command) แก้ไขพื้นที่รอบตัวได้เล็กน้อย
4.  **Admin (แอดมิน):** ดีบักเกอร์ระดับสูง เปลี่ยนกฎฟิสิกส์ใน Dead Sector ได้
5.  **Dev (เดฟ - ตำนาน):** ตัวตนระดับตำนานที่สามารถ "เขียน" ความจริงใหม่ได้ (ยังไม่ปรากฏ)
6.  **Anomaly (อะนอมอลี):** ตัวเอก (พระเอก) - ไม่ได้อยู่ในระบบ แต่เป็น "ไวรัส" ที่ระบบพยายามกำจัดแต่ทำไม่ได้

## 4. ตัวละครหลัก (Characters) & ชื่อ (Naming Convention)
ใช้ชื่อสไตล์ญี่ปุ่นสมัยใหม่ (Modern Edgy) ให้ความรู้สึกเหมือนตัวละครในโลกไซเบอร์หรืออนิเมะดาร์กๆ

### ตัวเอก (Protagonist)
*   **ชื่อ:** **คุริโตะ (Kurito - クトリト)** (เปลี่ยนจาก Kaito)
    *   *ฉายา:* The Null Pointer (ผู้มีค่าเป็นศูนย์), Zero
    *   *สถานะปัจจุบัน:* **Level 0 (Guest)** - ต่ำกว่า User ทั่วไป เป็นเพียงคนแบกของที่ไม่มีใครต้องการ
    *   *แรงจูงใจ:* ต้องการหาเงินและ **"Source Fragments"** บริสุทธิ์มารักษา **"อาการ Glitch Syndrome"** ของน้องสาว
    *   *ความสามารถลับ:* **[Execute Code]** - สามารถรันคำสั่งพิเศษที่คนอื่นทำไม่ได้ แต่ตอนนี้ยังเป็นความลับเพราะไม่มี Mana (Resource)

### ตัวละครสำคัญ (Key Characters)
*   **ไอริ (Airi):** น้องสาวของคุริโตะ ป่วยเป็นโรคประหลาดที่ร่างกายค่อยๆ จางหายไปเหมือนไฟล์ที่ถูกลบ (Glitch Syndrome)
*   **เร็น (Ren):** คู่แข่ง/เพื่อนสมัยเด็ก ระดับ Admin สายความเร็ว สังกัด Server ใหญ่ระดับประเทศ "Neo-Tokyo"
*   **อาสึมิ (Azumi):** หัวหน้า Server หญิงผู้เข้มงวด เชี่ยวชาญการป้องกัน (Firewall)
*   **ชิน (Shin):** รุ่นน้องผู้มองโลกในแง่ดี (User Level) เป็นคนเดียวที่ยังคุยกับคุริโตะดีๆ
*   **โกลบอลสเกล (Global Scale):** เนื้อเรื่องจะขยายไปสู่ระดับโลก โดยมี "Server" (กิลด์) ใหญ่ๆ ควบคุมแต่ละทวีป เช่น US-East, EU-Central, Asia-Pacific

## 5. จุดเริ่มต้นเรื่อง (Prologue Concept)
คุริโตะคือ "Guest" (ระดับ 0) ที่จนตรอก ต้องรับจ้างแบกของใน Dead Sector ระดับต่ำเพื่อหาเงินซื้อ "Stabilizer" ประทังชีวิตน้องสาวที่กำลังจะ "Error" หายไป วันหนึ่งปาร์ตี้ของเขาดันไปเปิดเจอ "Hidden Directory" (ดันเจี้ยนลับ) ที่ระดับความอันตรายประเมินค่าไม่ได้... และนั่นคือจุดเริ่มต้นที่เขาจะได้เห็น "Source Code" ของพระเจ้า
